Цель:
Удерживать активность постоянных клиентов с помощью персонализированных предложений.

Задачи:
1. Построить модели, которые предскажут вероятность снижения покупательской активности клиента в следующие три месяца.
2. Выбрать лучшую модель.
3. В исследование нужно включить дополнительные данные финансового департамента о прибыльности клиента: какой доход каждый покупатель приносил компании за последние три месяца.
4. Используя данные модели и данные о прибыльности клиентов, выделить сегменты покупателей и разработать для них персонализированные предложения.


Используемые модели:
1.  KNeighborsClassifier
2.  DecisionTreeClassifier
3.  LogisticRegression
4.  SVC


Результат:
Лучшей моделью является LogisticRegression(C=6, penalty='l1', random_state=42, solver='liblinear').

Для бизнеса важно не упустить выгоду от клиентов. Тогда ему важно определить всех тех, чья активность снизится, чтобы стимулировать таких клиентов к покупкам. В таком случае для модели важна метрика recall. Она отвечает за то правильно ли присвоен всем объектам класс 1, то есть активность снизится. Для такой модели важно сократить ошибки второго рода – предсказать снижение активности.

recall = 0,886. То есть модель в 88,6% случаев верно определила, что активность клиента снизится, но в 11,4% случаев она ошибается и предсказывает, что активность клиента не снизится, хотя в действительности она снизалась.

Оптимальный порог для построения предсказаний модели равен 0,82. То есть при прогнозировании активности покупателей, будем использовать этот порог как оптимальный для предсказания. Он позволит уменьшить количество ошибок.

Признаки, которые сильнее влияют на целевой признак:

- минут текущий месяц
- неоплаченные продукты штук квартал
- минут предыдущий месяц
- популярная категория
- страниц за визит
- средний просмотр категорий за визит 
- маркет актив 6 мес
